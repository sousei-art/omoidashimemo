# 思い出しメモ

## 概要

「思い出しメモ」は、忘れやすい小ネタ・手順・持ち物情報をすぐ登録し、あとから素早く検索するための個人用Webアプリです。

主な用途は以下の2つです。

1. 小ネタ・手順メモ
   - 一度調べた方法
   - ChatGPTに教えてもらったレシピ
   - iPhoneやアプリの操作手順
   - 仕事や生活のちょっとしたノウハウ

2. 持ち物情報
   - 家電、ガジェット、工具、スポーツ用品などの情報
   - メーカー、型番、購入日、保証期限、保管場所
   - 所持中、故障、紛失・廃棄、売却済みの状況管理
   - 写真や説明書URLの記録

## 形式

空手試合管理アプリに近い、単体Webアプリ形式です。

- HTML / CSS / JavaScript で構成
- ZIPで受け渡し可能
- GitHubにアップロード可能
- Vercelで公開可能
- iPhone Safari / Android Chrome / PCブラウザで利用可能
- PWA風にホーム画面追加可能

---

## 保存方式

データは端末内の IndexedDB に保存します。

- 小ネタメモ：IndexedDBに保存
- 持ち物情報：IndexedDBに保存
- 写真データ：圧縮後にIndexedDBへ保存
- バックアップ：JSONファイルとして書き出し可能
- 復元：JSONファイルから読み込み可能

---

## 重要な注意事項

ブラウザのデータ削除、端末変更、Safariの履歴・Webサイトデータ削除などを行うと、アプリ内の保存データが消える可能性があります。

そのため、定期的に `JSONを書き出す` でバックアップを取ってください。

パスワード、クレジットカード番号、マイナンバーなどの機密情報は登録しないでください。
写真もJSONバックアップに含まれます。

## 主な機能

### 1. 小ネタメモ

忘れやすい手順や知識を登録できます。

登録項目：

- タイトル
- カテゴリ
- タグ
- 本文
- 手順
- 注意点
- 参考URL
- お気に入り
- 登録日
- 更新日

---

### 2. 持ち物情報

自分の持ち物を台帳として登録できます。

登録項目：

- 品名
- カテゴリ
- 所持状況
- メーカー
- 型番
- 写真
- 購入日
- 購入店
- 価格
- 保証期限
- 保管場所
- 説明書URL
- 消耗品メモ
- 自由メモ
- タグ
- お気に入り
- 登録日
- 更新日

所持状況は以下から選択します。

- 所持中
- 故障
- 紛失・廃棄
- 売却済み

---

### 3. 検索

小ネタメモと持ち物情報をまとめて検索できます。

検索対象：

- タイトル
- 品名
- カテゴリ
- タグ
- 本文
- 手順
- 注意点
- 所持状況
- メーカー
- 型番
- 保管場所
- メモ
- 購入日
- 保証期限

---

### 4. 絞り込み・並び替え

以下の条件で絞り込みできます。

- 小ネタ
- 持ち物
- カテゴリ
- タグ
- お気に入り表示
- 持ち物の所持状況

持ち物一覧では以下の並び替えができます。

- 購入日 新しい順
- 購入日 古い順
- 更新日 新しい順

購入日が未入力の持ち物は、購入日順では一覧の下に表示されます。

---

### 5. バックアップ

保存データをJSON形式で書き出せます。

バックアップ対象：

- 小ネタメモ
- 持ち物情報
- 所持状況
- 写真データ
- タグ
- カテゴリ

---

### 6. 復元

JSONバックアップファイルからデータを復元できます。

復元方法は2種類です。

- 追加で復元
  - 既存データを残して読み込みます。
- 全消去して復元
  - 現在のデータを削除してから読み込みます。

v1.0.0で書き出したバックアップを復元した場合、持ち物の所持状況は自動で「所持中」になります。

## ローカルで開く方法

### 直接開く

`index.html` をブラウザで開きます。

ただし、PWAやService Workerは `file://` では動作しません。
アプリ本体の登録・検索・バックアップ機能は利用できます。

### 簡易サーバーで開く

Pythonが使える場合は、フォルダ内で以下を実行します。

```bash
python3 -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000
```

## GitHub / Vercelで使う場合

1. このフォルダの中身をGitHubリポジトリへアップロード
2. Vercelでそのリポジトリを読み込み
3. Framework Preset は `Other` または静的サイトとして扱う
4. Build Command は不要
5. Output Directory は空欄またはルート

---

## 推奨運用

- 小ネタは「タイトル」と「タグ」を短く明確にする
- 持ち物は「型番」「購入日」「保管場所」「保証期限」を優先して入れる
- 壊れた物や手放した物は削除せず、所持状況を変更して履歴として残す
- 迷ったらカテゴリは「その他」でよい
- 後から探す単語をタグに入れる
- 写真を多用する場合はバックアップJSONのサイズが大きくなる
- 定期的にJSONバックアップを取る
