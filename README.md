# 思い出しメモ

## 概要

「思い出しメモ」は、忘れやすい小ネタ・手順・持ち物情報をすぐ登録し、あとから素早く検索するための個人用Webアプリです。

主な用途は以下の2つです。

1. 小ネタ・手順メモ
   - 一度調べた方法
   - ChatGPTに教えてもらったレシピ
   - iPhoneやアプリの操作手順
   - 仕事や生活のちょっとしたノウハウ
   - 画像・スクリーンショット付きの手順メモ

2. 持ち物情報
   - 家電、ガジェット、工具、スポーツ用品などの情報
   - メーカー、型番、購入日、保証期限、保管場所
   - 所持中（使用中／未使用／使用終わり）、故障、紛失・廃棄、売却済みの状況管理
   - 写真や説明書URLの記録

- 持ち物の所持状況を細分化
  - 所持中（使用中）
  - 所持中（未使用）
  - 所持中（使用終わり）
  - 故障
  - 紛失・廃棄
  - 売却済み

## 形式

空手試合管理アプリに近い、単体Webアプリ形式です。

- HTML / CSS / JavaScript で構成
- ZIPで受け渡し可能
- GitHubにアップロード可能
- GitHub Pagesで公開可能
- Vercelで公開可能
- iPhone Safari / Android Chrome / PCブラウザで利用可能
- PWA風にホーム画面追加可能

---

## 保存方式

データは端末内の IndexedDB に保存します。

- 小ネタメモ：IndexedDBに保存
- 小ネタ画像：圧縮後にIndexedDBへ保存
- 持ち物情報：IndexedDBに保存
- 持ち物写真：圧縮後にIndexedDBへ保存
- バックアップ：JSONファイルとして書き出し可能
- 復元：JSONファイルから読み込み可能

---

## 重要な注意事項

ブラウザのデータ削除、端末変更、Safariの履歴・Webサイトデータ削除などを行うと、アプリ内の保存データが消える可能性があります。

そのため、定期的にJSONバックアップを取ってください。

パスワード、クレジットカード番号、マイナンバーなどの機密情報は登録しないでください。
画像・写真もJSONバックアップに含まれます。

### 3. タグ入力

タグは以下の区切りで入力できます。

```text
半角スペース
全角スペース
カンマ
読点
改行
```

入力例：

```text
iPhone 再起動 不具合対応
```

この場合、以下の3タグとして保存されます。

```text
#iPhone
#再起動
#不具合対応
```

注意：タグ名の中にスペースは使えません。
`強制 再起動` と入力すると、`#強制` と `#再起動` に分かれます。
1つのタグにしたい場合は `強制再起動` と入力してください。

### 5. 絞り込み・並び替え

以下の条件で絞り込みできます。

---

### 6. バックアップ

保存データをJSON形式で書き出せます。

バックアップ対象：

- 小ネタメモ
- 小ネタ画像
- 持ち物情報
- 持ち物写真
- 所持状況
- タグ
- カテゴリ

書き出し方法：

- 保存先を選んでJSONを書き出す
  - 対応ブラウザでは保存先選択ダイアログを開きます。
  - iPhone Safariなど未対応環境では共有またはダウンロードに切り替わります。
- JSONを共有/ファイルに保存
  - iPhoneでは共有シートから「ファイルに保存」を選ぶ運用を想定しています。
- JSONをダウンロード
  - 従来通りブラウザのダウンロード動作で保存します。

---

### 7. 復元

JSONバックアップファイルからデータを復元できます。

復元方法は3種類です。

- 追加で復元
  - JSON内のデータを別データとして追加します。
  - 既存データは上書きしません。
- 差分復元
  - JSONにだけあるデータを追加します。
  - JSON側の更新日時が新しい同一IDデータだけ上書きします。
  - 端末側だけにあるデータは削除しません。
  - 端末側の方が新しいデータ、同時刻で内容差がある競合データはスキップします。
- 全消去して復元
  - 現在のデータを削除してから読み込みます。

v1.0.0〜v1.0.2で書き出したバックアップを復元した場合、持ち物の旧「所持中」は自動で「所持中（使用中）」になります。

## ローカルで開く方法

### 直接開く

`index.html` をブラウザで開きます。

ただし、PWAやService Workerは `file://` では動作しません。
アプリ本体の登録・検索・バックアップ機能は利用できます。

### 簡易サーバーで開く

Pythonが使える場合は、フォルダ内で以下を実行します。

```bash
python3 -m http.server 8000
```

ブラウザで以下を開きます。

```text
http://localhost:8000
```

---

## GitHub Pagesで使う場合

1. このフォルダの中身をGitHubリポジトリへアップロード
2. `Settings` → `Pages` を開く
3. `Source` を `Deploy from a branch` にする
4. `Branch` を `main`、フォルダを `/root` にする
5. `Save` を押す
6. 表示されたURLをiPhone Safariで開く
7. 共有ボタンから「ホーム画面に追加」を行う

---

## Vercelで使う場合

1. このフォルダの中身をGitHubリポジトリへアップロード
2. Vercelでそのリポジトリを読み込み
3. Framework Preset は `Other` または静的サイトとして扱う
4. Build Command は不要
5. Output Directory は空欄またはルート
6. Deploy する

## 推奨運用

- 小ネタは「タイトル」と「タグ」を短く明確にする
- 小ネタ画像は、手順のスクショや完成写真など探す助けになるものだけ入れる
- 持ち物は「型番」「購入日」「保管場所」「保証期限」を優先して入れる
- 壊れた物や手放した物は削除せず、所持状況を変更して履歴として残す
- 迷ったらカテゴリは「その他」でよい
- 後から探す単語をタグに入れる
- 画像・写真を多用する場合はバックアップJSONのサイズが大きくなる
- 定期的にJSONバックアップを取る
